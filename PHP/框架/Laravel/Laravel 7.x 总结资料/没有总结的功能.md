# 清单

## 一、核心架构
### 1.请求周期：理解了该工具的工作原理。
### 2.服务容器：管理类依赖以及实现依赖注入。
#### 2.1.依赖注入：通过构造函数将类依赖「注入」到类中。
### 3.服务提供者：服务提供者是所有 Laravel 应用程序的引导中心。
### 4.Facades：为应用的 服务容器 提供了一个「静态」 接口。
### 5.Contracts：契约是一组接口，它们由框架提供并定义了核心服务。

## 二、前端开发
### 1.编辑资源Mix：定义 Webpack 构建步骤的 API，前端预处理器。

## 三、安全相关
### 1.用户授权：处理用户的授权动作。
### 2.Email认证：发送和校验电子邮件的验证请求。
### 3.重置密码：重置密码的功能。

## 四、综合话题
### 1.广播系统：通过WebSocket连接使服务端和客户端之间数据更新。
### 2.集合：便于处理数组数据的封装
### 3.事件系统：简单的观察者实现，允许你在应用中订阅和监听各种发生的事件。
### 4.文件系统：文件存储与云存储，涉及文件上传与目录相关。
### 5.HTTP客户端：使用 HTTP 请求与其他 Web 应用进行通信，用于创建请求和模拟响应。
### 6.邮件发送：让你可以快速从本地或云端服务自由地发送邮件。
### 7.消息通知：除邮件发送，还支持短信、群发等多种频道发送通知。
### 8.扩展包开发：(composer)扩展包是向 Laravel 添加功能的主要方式。
### 9.队列：将耗时任务的处理往后推延。
### 10.任务调度：执行定时任务。

## 五、数据库
### 1.数据填充：为你的数据库填充测试数据
### 2.Redis：Redis配置以及使用

## 六、Model(Eloquent ORM)
### 1.模型关联：数据库表通常相互关联。如，文章可能有许多评论，订单对应一个下单用户。
### 2.Eloquent集合：处理模型返回的结果集
### 3.修改器：对 Eloquent 属性值进行格式化操作。如加密某个数据库中的值。
### 4.API资源：当构建 API 时，能够将模型和模型集合转化成 JSON。
### 5.序列化：构建 JSON API 时，经常需要把模型和关联转化为数组或 JSON，对序列化中的属性控制。

## 七、测试相关
### 1.快速入门：测试你的应用程序。
### 2.HTTP测试：用于向您的应用程序发出 HTTP 请求并检查输出。
### 3.命令行测试：测试用户输入的控制台应用。
### 4.数据库测试：测试数据库驱动。
### 5.测试模拟器Mocking：「模拟」应用程序的某些功能的行为。如对象、任务、邮件、通知、队列等。
